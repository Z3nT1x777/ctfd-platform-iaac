# Legacy Challenge Template

This directory is a legacy single-template scaffold.

Preferred source for new challenges is [challenges/_templates](../_templates), which provides family-specific templates.

## Legacy Usage

1. Copy this directory to `challenges/<new-challenge-name>`.
2. Update `challenge.yml` metadata.
3. Update runtime files (`Dockerfile`, `app.py`, `requirements.txt`, `docker-compose.yml`).
4. Validate structure with repository validation scripts.
5. Run with Docker Compose inside the VM.

## Recommendation

Use family templates instead of this legacy scaffold for all new work.
